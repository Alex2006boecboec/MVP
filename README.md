# MusicRent — MVP аренды портативных музыкальных плееров

Учебный MVP по аналогии с сервисом "Бери заряд", только вместо павербанков —
портативные музыкальные проигрыватели (Sony Walkman, FiiO и т.п.). Пользователь
находит ближайшую станцию на карте, берёт плеер, слушает музыку через свои
наушники и сдаёт устройство на любой станции сети.

## Технологический стек

| Слой         | Технология                                                  |
| ------------ | ----------------------------------------------------------- |
| Фронтенд     | Next.js 14 (App Router), React 18, TypeScript, TailwindCSS  |
| Карта        | React Leaflet + OpenStreetMap                               |
| Бэкенд / API | Next.js Route Handlers                                      |
| База данных  | PostgreSQL + Prisma ORM                                     |
| Авторизация  | JWT (jose) в `httpOnly`-cookie + bcryptjs                   |
| Валидация    | Zod                                                         |
| Деплой       | Railway (Nixpacks + Postgres plugin)                        |

## План реализации MVP

1. **Проектирование БД** — 4 сущности: `User`, `Station`, `Player`, `Rental`.
2. **Авторизация** — регистрация / логин через JWT, роль `USER`/`ADMIN`.
3. **Каталог станций** — главная страница с интерактивной картой + список.
4. **Аренда плеера** — транзакционный обработчик, чек на баланс и активную аренду.
5. **Возврат на любую станцию** — расчёт стоимости по часам, списание с баланса,
   обновление статуса плеера, заряд батареи уменьшается на 10%.
6. **Личный кабинет** — баланс, активная аренда, история, итого потрачено.
7. **Админка** — CRUD станций и плееров, статусы плееров (`AVAILABLE` / `MAINTENANCE`).
8. **Сидер** — 5 тестовых станций по Москве и плееры разных моделей.
9. **Деплой на Railway** — Postgres plugin + Nixpacks билд.

## Локальный запуск

### 1. Установить зависимости

```bash
npm install
```

### 2. Поднять Postgres (например, в Docker)

```bash
docker run --name musicrent-pg \
  -e POSTGRES_PASSWORD=postgres \
  -e POSTGRES_DB=musicrent \
  -p 5432:5432 -d postgres:16
```

### 3. Скопировать `.env.example` в `.env`

```bash
cp .env.example .env
```

Отредактируйте `DATABASE_URL` и обязательно задайте `JWT_SECRET`
(например, любая длинная случайная строка).

### 4. Применить схему и наполнить данными

```bash
npx prisma db push
npm run db:seed
```

### 5. Запустить dev-сервер

```bash
npm run dev
```

Откройте [http://localhost:3000](http://localhost:3000).

## Деплой на Railway

1. **Создать проект** на [railway.com](https://railway.com).
2. **Подключить репозиторий** (`New → Deploy from GitHub repo`).
3. **Добавить плагин Postgres**: в проекте `+ New → Database → Postgres`.
   Railway автоматически прокинет `DATABASE_URL` в сервис.
4. **Задать переменные окружения** для сервиса приложения:
   - `JWT_SECRET` — длинная случайная строка (обязательно)
   - `ADMIN_EMAIL` — email для админа (по умолчанию `admin@musicrent.local`)
   - `ADMIN_PASSWORD` — пароль админа
   - `NODE_ENV=production`
5. **Запустить деплой**. Railway возьмёт Nixpacks-конфиг,
   установит зависимости, выполнит `prisma db push`, соберёт Next.js
   и засидит БД тестовыми данными.
6. **Сгенерировать публичный домен** в настройках сервиса
   (Settings → Networking → Generate Domain).

## Сценарии демонстрации преподавателю

1. Регистрация нового пользователя → начислено 500 ₽ на баланс.
2. Главная: показать карту со станциями (Leaflet, OpenStreetMap).
3. Открыть станцию, взять плеер → статус "В аренде", баланс на месте.
4. Профиль: показать активную аренду.
5. Открыть другую станцию → сдать плеер → списание по часам,
   плеер появился на новой станции.
6. История в профиле.
7. Войти под админом → CRUD станций и плееров,
   перевод плеера в обслуживание.

## Что можно добавить дальше (вне MVP)

- Платежи (Stripe / ЮKassa) для пополнения баланса.
- QR-код станции/плеера и сканер в браузере.
- Push-уведомления о просрочке аренды.
- Скидка/абонемент на длительные аренды.
- Геолокация пользователя и сортировка станций "ближайшие ко мне".
- Загрузка кастомных плейлистов на плеер через карточку плеера.
- Тарифы по моделям плееров (премиум-устройства дороже).
- Тесты: Vitest для бизнес-логики (`calculateCost`), Playwright для e2e-сценариев.
